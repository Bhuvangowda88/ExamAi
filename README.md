# Interview Assistant (Electron)

A desktop interview assistant built with Electron, React, TypeScript, and Gemini.

This app helps you capture interview questions (voice or typed input) and generate concise, professional answers in real time.

## Highlights

- Floating always-on-top desktop window
- Voice input using Web Speech API
- Text input mode with manual Generate Answer action
- Gemini answer generation through Electron IPC (main process)
- Frameless compact UI with pin, hide, opacity, and tray controls
- Global keyboard shortcuts for quick use
- macOS DMG build support

## Tech Stack

- Electron 28
- electron-vite
- React 18
- TypeScript
- Tailwind CSS
- Google Generative AI SDK

## Project Structure

- src/main: Electron main process and IPC handlers
- src/preload: Secure bridge API exposed to renderer
- src/renderer: React UI
- out: Build output generated by electron-vite
- release: Installer artifacts generated by electron-builder

## Prerequisites

- Node.js 18+ (recommended Node.js 20+)
- npm
- Gemini API key with Generative Language API access

## Setup

1. Install dependencies

   npm install

2. Add your Gemini key

   The project reads a default renderer env key from:

   src/renderer/.env.local

   Add:

   VITE_GEMINI_API_KEY=YOUR_GEMINI_API_KEY

   You can also set or override the key from the in-app Settings panel. It is stored in localStorage.

## Run in Development

Start Electron + Vite dev mode:

npm run dev

Notes:

- Vite may auto-select another port if 5173 is occupied.
- Use the newly opened Electron dev window after restart.

## Build

Create production build output:

npm run build

This generates compiled files under out/.

## Package for macOS

Build DMG installer:

npm run build:mac

Artifacts are placed in release/.

## App Behavior and Controls

### Input modes

- Voice mode: Use Start to begin speech recognition.
- Type mode: Click Type, enter a question, then click Generate Answer.

### Global shortcuts

- CmdOrCtrl+Shift+H: Show or hide window
- CmdOrCtrl+Shift+R: Toggle recording
- CmdOrCtrl+Shift+Escape: Quick hide

## Gemini Model

Main process uses:

- gemini-2.5-flash

If you change model names in code, ensure the model exists for your account and supports generateContent.

## Troubleshooting

### Network error shown while internet is fine

Possible cause: model mismatch or old running process.

Fix:

1. Stop all old Electron/dev processes
2. Rebuild and restart:

   npm run build
   npm run dev

3. Verify the active process is using the latest code

### 404 model not found

If logs mention a model not found error, your runtime is likely using an unsupported or stale model name.

Current supported default in this project is gemini-2.5-flash.

### Speech recognition network error

Speech recognition depends on browser speech services and mic permissions.

- Grant microphone access in macOS Privacy settings
- If speech fails, use Type mode as fallback

### API key issues

If generation fails with auth/key errors:

- Confirm key is valid
- Confirm API is enabled in Google Cloud project
- Re-save key in Settings and retry

## Security Notes

- API key entered in Settings is stored locally in browser localStorage.
- Do not commit personal keys to public repositories.
